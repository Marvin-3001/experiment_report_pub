# 实验数据绘图与报告生成工具

这是一个面向仪器分析实验的桌面应用，使用 PyQt6 提供图形界面，将实验数据绘图、曲线拟合、实验报告填写和 A4 PDF
导出整合在同一个程序中。项目内置 8 类实验报告模板和 8 个绘图入口，也支持在程序目录外部扩展自定义 JSON 报告模板。

## 主要功能

### 1. 实验数据绘图

- 按实验类型自动创建对应的数据表格、列标题和配色。
- 支持直接编辑表格，也支持从 Excel 复制多行、多列数据后使用 `Ctrl+V` 粘贴。
- 绘图表格可按输入内容自动增加行，空白行不会参与计算。
- 图标题、X 轴标题和 Y 轴标题均可自定义；留空时使用实验配置中的默认标题。
- 在窗口中即时预览生成的图像。
- 可将图像导出为 PNG、JPG、SVG 或 PDF，导出分辨率为 300 DPI。
- 可将当前图像加入报告，生成 `{picture1}`、`{picture2}` 等图片变量。

当前支持的绘图实验如下：

| 实验                   | 数据列                    | 绘图或拟合方式                         |
|----------------------|------------------------|---------------------------------|
| 离子选择性电极测定氟离子         | lgC F-、Voltage         | 一元线性拟合                          |
| 甲苯、萘的高效液相色谱分析        | Concentration、Response | 一元线性拟合                          |
| 气相色谱流动相速度对柱效的影响      | u、H                    | 简化范氏方程 `H = A + B/u + Cu` 非线性拟合 |
| ICP-OES 多元素同时测定      | Concentration、Signal   | 一元线性拟合                          |
| 火焰原子吸收测定水样中的钾        | Concentration、Signal   | 一元线性拟合                          |
| 原子荧光测定天然水中的砷和汞       | Concentration、Signal   | 一元线性拟合                          |
| 吖啶橙荧光探针法测定 DNA—标准曲线图 | Concentration、Signal   | 一元线性拟合                          |
| 吖啶橙荧光探针法测定 DNA—光谱图   | Wavelength、Intensity   | 连续折线光谱图                         |

线性拟合图会显示回归方程和决定系数 R²；范氏方程拟合图会显示 A、B、C 的拟合结果和 R²。

### 2. 实验报告填写

- 根据所选 JSON 模板动态生成填写表单。
- 支持单行文本、多行文本、日期、图片选择和可编辑网格表格等字段类型。
- 必填字段会显示 `*`，生成 PDF 前统一校验。
- 日期字段提供日历选择器，默认使用当天日期。
- 网格表格支持固定单元格、可编辑单元格、合并单元格、自定义行高和列宽。
- 网格表格支持从 Excel 粘贴数据、清空可编辑单元格，并保护模板中的只读内容。
- 右侧提供纯文本报告预览和当前可用图片变量提示。
- 支持一次选择并导入多张 PNG、JPG、JPEG 或 BMP 图片，依次生成 `{import_picture1}`、`{import_picture2}` 等变量。

项目内置以下报告模板：

1. 原子荧光法测定天然水中的砷和汞
2. 循环伏安法测电极反应参数
3. 吖啶橙荧光探针法测定 DNA
4. 离子选择性电极测定氟离子
5. 流动相速度对柱效的影响
6. 甲苯、萘的高效液相色谱分析
7. ICP-OES 的多元素同时测定
8. 水样中钾的原子吸收法测定

各模板已经包含相应实验的目的、原理、仪器与试剂、实验步骤、结果处理和思考题结构。部分模板还包含预设示意图、三线表或需要填写的实验数据表格。

### 3. PDF 报告生成

- 输出标准 A4 PDF，页面四周设置适合实验报告的页边距。
- 自动生成报告标题、基本信息表和模板定义的各级正文内容。
- 支持中文与西文混排：中文使用 ReportLab 的 `STSong-Light`，英文、数字、常用单位和数学符号使用 Times New Roman。
- 支持正文首行缩进、上下标、换行和缺失内容占位线。
- 支持三线表、可填写网格表、合并单元格、预设图片、绘图图片和外部导入图片。
- 图片会在不放大的前提下按页面可用区域等比例缩放并居中。
- 默认文件名取自“实验名称”，Windows 文件名中的非法字符会自动替换为下划线。

### 4. 外部模板扩展

程序会每秒检查一次程序所在目录下的 `customer_templates` 文件夹，并自动发现其中新增、删除或修改的 `.json`
文件。合法模板会出现在“外部模板”下拉框中；格式错误、名称重复等问题会在界面中提示，并且不会影响其他有效模板。

模板正文支持以下常用写法：

```text
{{field_key}}                 将表单字段填入正文
[indent=21]正文               设置 PDF 段落首行缩进
10[sup=-3]、E[sub=pa]         上标和下标
{picture1}                    插入绘图页加入的图片
{import_picture1}             插入手动导入的图片
```

图片占位符也兼容双大括号写法，例如 `{{picture1}}`。模板还可在 `sections.content` 中配置 `three_line_table`、`preset_image`
和 `grid_table` 对象。预设图片必须放在 `templates/image` 中，模板的 `file` 属性只能填写文件名，不能填写子目录或 `../` 路径。

## 环境要求

- Python 3.11 或更高版本（以 `requirements.txt` 中声明的运行时版本为准）
- Windows 桌面环境（项目会优先从 Windows 字体目录查找 Times New Roman）
- 系统已安装 Times New Roman 字体

`requirements.txt` 已固定以下运行依赖版本：

| 依赖         | 版本     |
|------------|--------|
| Matplotlib | 3.10.8 |
| NumPy      | 2.4.4  |
| Pillow     | 12.2.0 |
| PyQt6      | 6.11.0 |
| ReportLab  | 4.5.0  |
| SciPy      | 1.17.1 |

可在虚拟环境中一次性安装全部运行依赖：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

## Windows EXE 版本

不希望配置 Python 环境的 Windows
用户，可以前往公开仓库的 [GitHub Releases](https://github.com/Marvin-3001/experiment_report_pub/releases) 页面下载已经打包好的
EXE 文件。

EXE 版本使用 PyInstaller 打包，包含程序运行所需的 Python 依赖和内置 `templates` 资源。下载后无需安装 Python，双击 EXE 即可启动。

使用 EXE 时请注意：

- EXE 仅面向 Windows；源码运行方式仍适用于需要修改代码、模板或进行二次开发的用户。
- 程序仍依赖系统中的 Times New Roman 字体；缺少该字体时可能无法正常启动或导出 PDF。
- 如需使用自定义报告模板，请在 EXE 所在目录创建 `customer_templates` 文件夹，并将模板 JSON 放入其中。

## 启动方式

以下方式适用于从源码运行。在项目根目录执行：

```powershell
python main.py
```

启动后主窗口包含两个标签页：

- `实验数据绘图`：录入数据、拟合或绘图、导出图片、把图片加入报告。
- `实验报告生成`：选择模板、填写报告、导入图片、刷新预览和生成 PDF。

## 推荐使用流程

1. 打开“实验数据绘图”，选择实验类型。
2. 从 Excel 复制数据，在左侧表格的目标起始单元格按 `Ctrl+V`。
3. 检查数据并按需修改图标题和坐标轴标题，然后点击“绘图”。
4. 如需把图放进报告，点击“加入报告图片”，记下界面提示的图片变量。
5. 切换到“实验报告生成”，选择对应的内置或外部模板。
6. 填写基本信息、实验结果和思考题；如需额外图片，点击“导入外部图片”。
7. 确保所用模板正文包含对应的图片变量，然后点击“刷新预览”。
8. 点击“生成 PDF”，补齐程序提示的必填项并选择保存位置。
9. 打开生成的 PDF，检查分页、表格、图片和特殊符号显示是否符合提交要求。

## 使用注意事项

### 数据与拟合

- 绘图表格中的每个非空数据行都必须包含所需的全部数值，不能混入单位、中文说明或其他非数字字符，否则程序会按行提示错误。
- 完全空白的行会被忽略。建议粘贴前确认小数点格式，并删除 Excel 中的标题行。
- 一元线性拟合至少需要 2 个有效数据点。
- 范氏方程拟合至少需要 3 个有效数据点，且流速 `u` 必须大于 0；数据分布不合理时，SciPy 也可能无法收敛。
- R² 是根据当前输入数据计算的，程序不会自动判断异常值、有效数字、单位或实验数据是否科学合理。
- “导出图片”保存的是当前画布；修改表格后应重新点击“绘图”，再导出或加入报告。

### 表单与预览

- 切换内置或外部模板会重新创建表单，当前未保存的填写内容会丢失。
- 程序目前不提供项目文件保存、自动保存或历史记录功能；关闭程序后表单内容不会恢复。长篇实验分析和思考题建议先在其他文档中留存副本。
- “刷新预览”不会检查必填项；“生成 PDF”才会执行完整的必填校验。
- 右侧预览是纯文本结构预览，不等同于最终 PDF 的字体、表格、图片和分页效果，提交前必须检查实际 PDF。
- 模板中的空字段在预览或正文中可能显示为 `________`，但标记为必填的字段仍无法在导出时留空。

### 图片

- “加入报告图片”只有在成功绘图后才可使用。
- 绘图图片和导入图片会复制到系统临时目录，图片变量只保存在当前程序进程中；重启程序后需要重新绘图或导入，变量编号也会重新开始。
- 图片变量不会自动插入任意报告位置，必须由当前模板正文引用对应占位符。
- 导入图片仅支持 PNG、JPG、JPEG 和 BMP；绘图单独导出时还支持 SVG 和 PDF。
- 删除、清理系统临时文件或移动相关资源后，尚未生成的报告可能找不到图片。

### 模板、字体与导出

- 内置模板来自 `templates` 根目录下的 JSON 文件；`templates/plot_configs` 中的 JSON 只负责绘图实验配置，不会作为报告模板加载。
- 外部模板目录名必须是 `customer_templates`，并与源码目录或打包后的 EXE 位于同一级。该目录不存在时程序仍可正常使用内置模板。
- 外部模板的 JSON 顶层必须是对象；建议为每个模板设置唯一且非空的 `template_name`。
- 缺少 Times New Roman 时，程序可能在启动阶段直接报错。中文绘图字体会依次尝试宋体、微软雅黑、黑体或 Noto Sans CJK
  SC；系统缺少合适中文字体时，图中的中文可能显示异常。
- 预设图片文件不可缺失或改名，否则引用它的报告无法生成。
- 若目标 PDF 已被阅读器或其他程序锁定，请先关闭占用程序再导出。

## 项目结构

```text
experiment_report/
├─ main.py                         程序入口
├─ desktop.py                      PyQt6 界面、表格交互和业务流程
├─ core.py                         模板加载、字体、占位符、图片状态等公共逻辑
├─ plotting.py                     数据校验、拟合、绘图和图片保存
├─ report.py                       报告预览及 ReportLab PDF 生成
├─ requirements.txt               Python 版本说明和固定版本的运行依赖
├─ templates/
│  ├─ *.json                       8 个内置实验报告模板
│  ├─ image/                       报告模板引用的预设图片
│  └─ plot_configs/
│     └─ plot_experiments.json     绘图实验、数据列和界面样式配置
└─ LICENSE                         GNU GPL v3 许可证
```
